## How to get these files into `Marketing Assets/`

This sandbox blocks outbound requests to `cloudfront.net` at the network policy level
(confirmed via `curl -sS http://127.0.0.1:44391/__agentproxy/status` → `403 policy denial`).
Options:
1. Allowlist `d8j0ntlcm91z4.cloudfront.net` (and `d2ol7oe51mr4n9.cloudfront.net`, used for
   the source reference images) in this environment's egress policy, then re-run the
   download step.
2. Download the 20 URLs above directly (browser or local machine, outside this sandbox)
   and drop them into `Marketing Assets/images/` and `Marketing Assets/videos/`.
3. Pull them from the Higgsfield web app / library (higgsfield.ai) — all 20 generations
   are saved there under this account's image/video history.

Link-rot note: one-time top-up credits and generated media are typically retained for the
account's active period — if these URLs expire, the same jobs can be re-fetched via the
Higgsfield MCP `job_display` tool using the job IDs embedded in each URL.
